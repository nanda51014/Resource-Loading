1  Load activities

Input: your activity list (Excel) with columns Activity ID, Activity Name, BOQ Code (Rak-BOQ), Activity Code, Qty, Unit — Start optional. Header row is auto-detected; WBS band rows and activities without an activity code are ignored. Durations are calculated by the tool at ONE crew (Qty ÷ Rate/day) — any Finish dates or durations in the input are overridden. Start date, if present, is used only to time-phase the histograms. Add crews later in P6 only where a duration must be compressed — your call.

5  Downloads

Durations file: Qty ÷ Rate/day at one crew — raw value and rounded-up whole workdays, for setting P6 original durations. The P6 file follows the standard P6 “Export to Excel” resource-assignment layout (sheets TASKRSRC + RSRC, internal field names in row 1, captions in row 2). Budgeted Units = crew make-up × calculated duration (raw) × hours/day, at one crew. Verify the first import against a copy of your project; if your P6 template expects different columns, export a blank TASKRSRC from your P6 and match it.
